# lf-test Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `lf-test`, a unified CLI + Go library for end-to-end testing of LiveForge across all protocols, including AV analysis, auth testing, and multi-cluster orchestration.

**Architecture:** A shared `tools/testkit/` Go library provides push, play, analysis, auth, and cluster engines. A cobra-based CLI (`tools/lf-test/`) exposes subcommands. Deterministic pre-encoded FLV media is embedded via `go:embed` and looped at runtime. All analysis operates on `*avframe.AVFrame` streams, fully decoupled from transport.

**Tech Stack:** Go stdlib, `cobra` (CLI), `pion/webrtc/v4` (WebRTC — already in go.mod), `datarhei/gosrt` (SRT — already in go.mod), `emiago/sipgo` (SIP/GB28181 — already in go.mod). Reuses existing LiveForge packages: `pkg/muxer/*`, `pkg/rtp`, `pkg/sdp`, `pkg/codec/*`, `module/rtmp`, `module/cluster`.

**Spec deviation:** `Pusher.Push()` returns `(*report.PushReport, error)` instead of the spec's `error`-only return. This is an intentional improvement — the pusher naturally has access to frame/byte counts and should report them directly rather than requiring a side-channel.

**Spec:** `docs/superpowers/specs/2026-04-04-lf-test-design.md`

import (
    "testing"
    "github.com/im-pingo/liveforge/pkg/avframe"

func TestDemuxerRoundTrip(t *testing.T) {
    // Create test frames
    videoSeq := avframe.NewAVFrame(avframe.MediaTypeVideo, avframe.CodecH264,
        avframe.FrameTypeSequenceHeader, 0, 0, []byte{0x01, 0x64, 0x00, 0x28, ...sps/pps...})
    audioSeq := avframe.NewAVFrame(avframe.MediaTypeAudio, avframe.CodecAAC,
        avframe.FrameTypeSequenceHeader, 0, 0, []byte{0x12, 0x10})
    kf := avframe.NewAVFrame(avframe.MediaTypeVideo, avframe.CodecH264,
        avframe.FrameTypeKeyframe, 0, 0, []byte{0x65, 0x88, 0x00, 0x01})
    af := avframe.NewAVFrame(avframe.MediaTypeAudio, avframe.CodecAAC,
        avframe.FrameTypeInterframe, 23, 23, []byte{0xFF, 0x01})

    // Mux
    m := NewMuxer(avframe.CodecH264, avframe.CodecAAC)
    initSeg := m.Init(videoSeq, audioSeq, 640, 360, 44100, 2)
    mediaSeg := m.WriteSegment([]*avframe.AVFrame{kf, af})

    // Demux
    d, err := NewDemuxer(initSeg)
    if err != nil {
        t.Fatalf("NewDemuxer: %v", err)
    }
    frames, err := d.Parse(mediaSeg)
    if err != nil {
        t.Fatalf("Parse: %v", err)
    }

    // Verify: should have at least 1 video + 1 audio frame
    var hasVideo, hasAudio bool
    for _, f := range frames {
        if f.MediaType.IsVideo() { hasVideo = true }
        if f.MediaType.IsAudio() { hasAudio = true }
    }
    if !hasVideo || !hasAudio {
        t.Errorf("expected video+audio, got video=%v audio=%v", hasVideo, hasAudio)
    }
}

// pkg/muxer/fmp4/demuxer.go
package fmp4

type Demuxer struct {
    videoTrackID uint32
    audioTrackID uint32
    videoCodec   avframe.CodecType
    audioCodec   avframe.CodecType
    timescale    map[uint32]uint32 // trackID -> timescale
}

// NewDemuxer parses the init segment (ftyp+moov) to extract track info.
func NewDemuxer(initSegment []byte) (*Demuxer, error)

// Parse demuxes a media segment (moof+mdat) and returns AVFrames with
// reconstructed DTS/PTS in milliseconds.
func (d *Demuxer) Parse(mediaSegment []byte) ([]*avframe.AVFrame, error)
```

Implementation must:
- Parse moov→trak→tkhd to get track IDs
- Parse moov→trak→mdia→mdhd to get timescale per track
- Parse moov→trak→mdia→hdlr to identify video vs audio
- Parse moof→traf→tfhd for track ID + default sample duration/size/flags
- Parse moof→traf→tfdt for base decode time (v1 = 8-byte uint64)
- Parse moof→traf→trun for per-sample duration/size/flags. **Note:** composition_time_offset support is conditional on trun flags bit `0x000800`. The current muxer does NOT write this flag, so PTS == DTS for muxer-generated segments. However, the demuxer must check this flag and apply offsets when present (for third-party segments with B-frames).
- Extract payloads from mdat using calculated offsets

// tools/testkit/source/flv_source_test.go
package source

import (
    "io"
    "testing"
    "github.com/im-pingo/liveforge/pkg/avframe"
)

func TestFLVSourceFrameCount(t *testing.T) {
    src := NewFLVSource(0) // no duration limit
    info := src.MediaInfo()
    if info.VideoCodec != avframe.CodecH264 { t.Fatal("expected H264") }
    if info.AudioCodec != avframe.CodecAAC { t.Fatal("expected AAC") }

    count := 0
    for {
        _, err := src.NextFrame()
        if err == io.EOF { break }
        if err != nil { t.Fatal(err) }
        count++
        if count > 200 { t.Fatal("too many frames, no EOF?") }
    }
    // Golden: ~60 video + ~86 audio + 2 seq headers ≈ 148 frames
    if count < 100 || count > 200 {
        t.Errorf("unexpected frame count: %d", count)
    }
}

func TestFLVSourceLoopDTSMonotonic(t *testing.T) {
    src := NewFLVSourceLoop(3) // loop 3 times
    var lastDTS int64 = -1
    seqHeaderCount := 0
    for {
        f, err := src.NextFrame()
        if err == io.EOF { break }
        if err != nil { t.Fatal(err) }
        if f.FrameType == avframe.FrameTypeSequenceHeader {
            seqHeaderCount++
            continue
        }
        if f.DTS < lastDTS {
            t.Fatalf("DTS not monotonic: %d < %d", f.DTS, lastDTS)
        }
        lastDTS = f.DTS
    }
    // Seq headers emitted at start + each loop boundary = at least 6 (2 per loop × 3)
    if seqHeaderCount < 6 {
        t.Errorf("expected >= 6 seq headers across 3 loops, got %d", seqHeaderCount)
    }
}
```

    a := New()

    a.Feed(avframe.NewAVFrame(avframe.MediaTypeVideo, avframe.CodecH264,

type rtspPusher struct{}

func (p *rtspPusher) Push(ctx context.Context, src source.Source, cfg PushConfig) (*report.PushReport, error) {
    // 1. Parse URL → host:port, path
    // 2. TCP connect (net.DialTimeout)
    // 3. Build SDP from src.MediaInfo() using sdp.BuildFromMediaInfo(info, baseURL, serverAddr)
    // 4. Send: ANNOUNCE rtsp://host/path RTSP/1.0\r\nContent-Type: application/sdp\r\n...

    // 10. On stop: TEARDOWN

    // 1. Dial WebSocket: gorilla/websocket or nhooyr.io/websocket
    //    URL: ws://host/live/test.flv (convert http:// to ws://)

}

**io.Reader bridging for WebSocket:** The FLV demuxer expects a continuous `io.Reader`, but WebSocket delivers discrete messages. Solution: `io.Pipe()` — a goroutine reads WS messages and writes them to the pipe writer; the demuxer reads from the pipe reader. This handles framing transparently.
